# Supervised Machine Learning from Scratch

Building every ML algorithm from the ground up — no scikit-learn, no TensorFlow, no abstractions. Every gradient, every loss function, every normalization step is written by hand using only Python, NumPy, Pandas, and Matplotlib.

The goal is simple: if you can build it from scratch, you truly understand it.

This repository is the mathematical foundation half of a two-repository system. The companion repository [ML-with-Scikit-Learn](https://github.com/ather-ops/ML-with-Scikit-Learn) applies the same concepts using production-grade tools. Study this one first.

## What Has Been Covered

**Data Preprocessing** — raw CSV ingestion, missing value detection via `.isnull()`, duplicate row removal, and structured validation before modeling.

**Outlier Detection** — full IQR method implemented manually. Fence thresholds computed from quartiles as `Q1 - 1.5*IQR` and `Q3 + 1.5*IQR`. Values beyond fences are clipped rather than deleted, preserving sample size while removing distortion.

**Feature Scaling** — Min-Max normalization from first principles: `x_scaled = (x - x_min) / (x_max - x_min)`. Applied to both features and target variables before training.

**Linear Regression** — slope and intercept initialized manually. Prediction computed as `y_hat = m*x + b`. MSE evaluated on every epoch to track convergence.

**Gradient Descent** — partial derivatives derived analytically and implemented as NumPy operations. Parameter update rule: `theta = theta - lr * grad`. Applied iteratively until loss converges.

**Logistic Regression** — sigmoid activation, binary cross-entropy loss, ROC curve generated by sweeping decision thresholds over raw sigmoid outputs.

## Metrics Implemented

| Metric | Formula | Status |
|--------|---------|--------|
| Mean Squared Error | `(1/n) * sum((y - y_hat)^2)` | Done |
| Root MSE | `sqrt(MSE)` | Done |
| Mean Absolute Error | `(1/n) * sum(abs(y - y_hat))` | Done |
| R-Squared | `1 - SS_res / SS_tot` | In Progress |
| Binary Cross-Entropy | `-[y*log(p) + (1-y)*log(1-p)]` | Done |
| ROC / AUC | Threshold sweep over sigmoid outputs | Done |
| Confusion Matrix | TP / FP / TN / FN manual computation | Done |

## Upcoming

**Ridge Regression** — linear regression with L2 regularization. The cost function becomes `MSE + lambda * sum(theta^2)`, which penalizes large weights and prevents overfitting on correlated features. Lambda controls the strength of the penalty.

**Lasso Regression** — linear regression with L1 regularization. The cost function becomes `MSE + lambda * sum(abs(theta))`. Unlike Ridge, Lasso drives some weights to exactly zero, effectively performing feature selection during training.

**Polynomial Features** — non-linear transformations applied to features before regression, allowing the model to fit curves while keeping the optimization linear in parameters.

**Multiple Regression** — vectorized gradient descent using the full design matrix: `y_hat = X @ theta`. Extends single-variable regression to any number of features without changing the core update rule.

**Train-Test Split** — manual shuffle and index-based split, no sklearn. Understanding this operation from scratch makes cross-validation intuitive.

**all_models.py** — a single consolidated file that imports and runs every implemented model end-to-end on a common dataset. One file to see all algorithms side by side, compare their outputs, and verify they agree with each other where they should.

## Core Concepts

**Why implement from scratch at all.** Scikit-learn's `LinearRegression()` fits in one line. But when your model underperforms and you need to debug it, you need to know whether the problem is in the gradient step, the scaling, the initialization, or the loss function. From-scratch implementations make every one of those components visible and modifiable.

**Gradient descent is the engine of almost everything.** Linear regression, logistic regression, Ridge, Lasso, and neural networks all optimize via gradient descent or a variant of it. Understanding the update rule `theta = theta - lr * grad` once means understanding all of them.

**Regularization answers a specific question.** If your training loss is low but validation loss is high, the model has memorized the training data. Ridge and Lasso both add a penalty term to the cost function that discourages large weights, forcing the model to generalize. The difference is which norm they use and whether they perform feature selection.

**Normalization is not optional.** Gradient descent behaves badly on features at different scales — the optimizer spends most of its steps correcting for the scale mismatch rather than finding the minimum. Scaling all features to the same range before training is not a preprocessing nicety; it is a correctness requirement.

cd Machine-Learning-from-scratch
